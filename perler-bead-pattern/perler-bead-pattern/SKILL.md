# QQ拼豆施工图生成器

将任意图片转换为专业拼豆施工图，基于 MARD 221 标准色卡。

## ⚠️ 必须询问用户的问题（不可跳过）

**在生成图纸之前，必须先询问用户以下问题：**

### 问题1：是否去除背景底色？（条件询问）

**⚠️ 仅当图片同时满足以下全部条件时才展示此询问：**

1. **主体为封闭图形** — 主体轮廓完整闭合，无断裂
2. **边缘清晰** — 主体与背景有明确的边界，无模糊过渡
3. **边缘外为白色底色** — 主体周围区域为纯白色或接近白色的浅色

**判断方法：**
- 检查图片四周（上、下、左、右）的像素颜色
- 如果边缘区域主要为白色（RGB接近255,255,255），且主体边缘清晰
- 则认为符合抠底条件，展示询问

**如果图片不符合上述条件（例如：背景复杂、边缘模糊、背景非白色），则跳过此询问，不展示背景处理选项。**

> 符合条件时询问：这张图片有明显的白色背景，是否去除背景（抠底）？
> - **去除** → 图纸中只保留主体轮廓，背景区域不放拼豆
> - **保留** → 白色背景也会变成白色拼豆

### 问题2：是否限制网格尺寸？

> 是否限制网格尺寸？
> - **不限制** → 根据图片自动计算最佳尺寸（推荐），自动还原图片细节
> - **限制** → 请选择固定尺寸：

**固定网格尺寸方案：**

| 方案 | 尺寸 | 颗数 | 适合场景 |
|------|------|------|----------|
| 🏃 快速 | 40×40 | 1,600颗 | 简单图案、初学者、快速完成 |
| ⚖️ 均衡 | 60×60 | 3,600颗 | 一般图案 |
| 🎨 精细 | 80×80 | 6,400颗 | 需要保留细节的复杂图案 |
| 👑 极致 | 100×100 | 10,000颗 | 追求最高还原度（耗时较长）|

> 💡 **自动计算规则**：当选择"不限制"时，脚本会分析图片复杂度（颜色数量、边缘密度、细节丰富度），自动计算最佳网格尺寸。简单图案可能只有30×30，复杂图片可能达到120×120，优先还原图片细节。

## 核心原则

1. **主体必须像素化**：禁止柔边、渐变、笔刷纹理和照片质感，每格纯色一颗拼豆
2. **专业施工图版式**：标题 + 坐标编号 + 5×5辅助线 + 格内色号 + 底部大色卡图例
3. **智能抠底**：仅当图片符合条件时才询问是否去除背景

## 功能

1. **智能抠底检测**：分析图片是否符合抠底条件（封闭主体+清晰边缘+白色背景），符合条件才询问
2. **像素化处理**：强制 NEAREST 插值 + 后处理消除中间色，确保纯像素风格
3. **LAB 色彩空间匹配**：映射到 MARD 221 色卡
4. **智能网格计算**：根据图片复杂度自动计算最佳网格尺寸，还原图片细节
5. **生成专业施工图 PNG**：
   - 顶部标题：`QQ拼豆施工图 — {文件名}`
   - 色块网格（纯色无渐变，每格代表一颗拼豆）
   - 5×5 加粗参考线（每5格加粗，方便数格子定位）
   - 行列坐标编号（从1开始，顶部列号 + 左侧行号）
   - 格内颜色符号标注（A10、H7、M8 等短色号，带描边）
   - **底部色卡图例**（色块≥24px + 色号 + 所需数量，布局宽松清晰）
   - 底部统计：总豆数、宽高格数、颜色总数

## 执行流程（必须严格遵守）

### Step 0: 图片预分析（必须首先执行）

**在询问用户之前，必须先对图片进行预分析：**

使用 bash 工具运行以下 Python 代码检测图片是否符合抠底条件：

```python
from PIL import Image
import sys

def check_bg_remove_eligibility(image_path):
    """检测图片是否符合抠底条件"""
    img = Image.open(image_path).convert('RGB')
    w, h = img.size
    
    # 采样边缘像素（四周边缘各采样20个点）
    edge_pixels = []
    sample_count = 20
    
    # 上边缘
    for x in range(0, w, w // sample_count):
        edge_pixels.append(img.getpixel((x, 0)))
    # 下边缘
    for x in range(0, w, w // sample_count):
        edge_pixels.append(img.getpixel((x, h-1)))
    # 左边缘
    for y in range(0, h, h // sample_count):
        edge_pixels.append(img.getpixel((0, y)))
    # 右边缘
    for y in range(0, h, h // sample_count):
        edge_pixels.append(img.getpixel((w-1, y)))
    
    # 检查边缘是否主要为白色（RGB各通道 > 230）
    white_count = 0
    for r, g, b in edge_pixels:
        if r > 230 and g > 230 and b > 230:
            white_count += 1
    
    white_ratio = white_count / len(edge_pixels)
    
    # 判断条件：边缘白色比例 > 70%
    return white_ratio > 0.7

# 运行检测
eligible = check_bg_remove_eligibility(sys.argv[1])
print(f"ELIGIBLE={'true' if eligible else 'false'}")
```

**判断逻辑：**
- 如果输出 `ELIGIBLE=true`：图片符合抠底条件，展示"是否去除背景"询问
- 如果输出 `ELIGIBLE=false`：图片不符合条件，跳过背景处理询问，只询问网格尺寸

## 关键约束

### 画面要求（必须遵守）

1. **像素化主体**：禁止柔边、渐变、笔刷纹理和照片质感。每格纯色，代表一颗拼豆
2. **每格纯色**：不做混色或半透明，每个像素格只有一种纯色
3. **关键区域保留**：面部、眼睛、嘴巴、手部、道具等细节优先保留
4. **颜色受控**：避免过多相近色，阴影用2-4个相邻色阶表达
5. **安全边距**：主体边缘不贴图纸边界，四周预留2-5格安全边距
6. **正交俯视图**：禁止透视、倾斜或立体渲染
7. **细节还原**：当选择"不限制网格尺寸"时，优先还原图片细节，网格尺寸可超过100

### 施工图版式（必须遵守）

- **顶部标题**：`QQ拼豆施工图 — {文件名}` + 网格尺寸 + 颜色数
- **坐标编号**：从1开始，上方列号 + 左侧行号
- **5×5辅助线**：每5格使用较粗参考线（3px灰色），方便定位
- **格内色号**：每格中心标注短色号（如A10、H7、M8），带对比色描边
- **底部色卡图例**：⭐重点要求⭐
  - 色块尺寸 **≥24px**（清晰可辨，不要过小）
  - 色块 + 色号 + 所需数量，横向排列，每行6-8个
  - 图例标题「🎨 色卡图例」加粗显示
  - 所有使用中的颜色全部列出，不可截断
- **底部统计**：总豆数 + 宽高格数 + 颜色总数

### 技术约束

- **像素化**：强制NEAREST插值 + 后处理消除中间色
- **锐化**：UnsharpMask(radius=2, percent=150, threshold=3)
- **色数自适应**：根据图片实际色彩复杂度自动决定色板大小
- **颜色匹配**：CIELAB ΔE色差公式
- **输出稳定**：相同输入 → 相同输出

## 色卡体系

**MARD 221 标准色卡**，数据内置在 `references/mard221_palette.json`。

色号：A1-A26, B1-B26, C1-C26, D1-D26, E1-E26, F1-F26, G1-G26, H1-H26, M1-M15（共221色）
